### 1-1. 프로젝트 개발의 필요성

○ 대중교통 정보 접근성 개선

－ 아산시에서 버스와 지하철을 이용하는 사용자는 정류장별 경유 버스, 버스 노선, 지하철역 정보, 환승 가능 경로, 주변 편의시설 정보를 상황에 따라 빠르게 확인할 필요가 있다.

● 여러 교통 정보가 분산되어 있으면 사용자는 원하는 이동 정보를 얻기 위해 반복 검색을 해야 하므로, 정류장 검색부터 환승 정보 조회까지 하나의 프로그램에서 처리할 수 있는 구조가 필요하다.

○ 실제 사용 가능한 교통 정보 조회 프로그램 개발

－ 본 프로젝트는 단순 기능 확인용 프로그램이 아니라, 사용자가 콘솔 메뉴를 통해 직접 검색하고 결과를 확인할 수 있는 서버/클라이언트 기반 대중교통 정보 시스템을 목표로 한다.

● 서버는 공공데이터포털에서 확보한 CSV 기반 교통 데이터를 로드하고, 클라이언트는 사용자의 검색 요청을 TCP 소켓으로 서버에 전달한다. 서버는 요청에 따라 정류장, 버스 노선, 지하철역, 환승 경로, 편의시설, 운행 정보를 처리하여 응답한다.

○ 개발 범위의 명확화

－ 서비스 범위는 실시간 위치 추적이나 외부 API 기반 실시간 혼잡도 집계가 아니라, 정적 CSV 데이터와 테스트용 운행 데이터를 기반으로 한 안정적인 조회 프로그램으로 정의한다.

● 채팅 기능과 학생정보관리 기능은 프로젝트 범위에서 제외하고, 버스와 지하철 기반의 교통 정보 조회 기능에 집중한다.

### 1-2. 프로젝트의 목표

○ 최종 개발 목표

－ 사용자가 실행 가능한 형태의 C 기반 대중교통 정보 조회 프로그램을 완성한다.

● 서버 접속 실패, 빈 검색어, 검색 결과 없음, 데이터 파일 누락, 클라이언트 비정상 종료 등 실제 사용 중 발생할 수 있는 상황을 고려하여 안정적으로 동작하도록 구현한다.

○ 주요 기능 목표

－ 정류장 검색, 버스 번호 검색, 지하철역 검색, 환승 정보 조회, 주변 편의시설 조회 기능을 제공한다.

● 사용자는 최종 메뉴에서 원하는 기능을 선택하고, 키워드나 번호를 입력하여 필요한 교통 정보를 조회할 수 있다.

○ 사용자 편의 기능 목표

－ 즐겨찾기와 최근 검색 기록을 클라이언트 로컬 파일로 관리하여 반복 사용성을 높인다.

● 자주 찾는 정류장, 버스 번호, 지하철역을 즐겨찾기로 저장하고, 최근 검색 기록 메뉴에서 이전 조회 내용을 확인할 수 있도록 한다.

○ 운행 정보 표시 목표

－ 테스트 운행 데이터를 활용하여 도착 예정시간, 혼잡도, 운행 상태를 표시한다.

● 실시간 API를 사용하지 않는 대신 `arrival_test.csv`, `crowding_test.csv`와 같은 보조 데이터를 활용하여 시연 가능한 운행 정보를 제공한다.

○ 운영 및 검증 목표

－ 서버 로그를 기록하여 클라이언트 접속, 요청 타입, 처리 결과, 오류 발생 내역을 확인할 수 있게 한다.

● 로그 파일은 `logs/server.log`에 저장하며, 서버 콘솔에도 주요 처리 내용을 출력한다.

○ 최종 사용자 메뉴

－ 클라이언트는 다음 메뉴 구성을 기준으로 개발한다.

● 메뉴 번호와 실제 요청 타입이 일치하도록 구현하여 사용자가 기능을 혼동하지 않도록 한다.

```text
===== 아산시 대중교통 정보 시스템 =====

1. 정류장 검색
2. 버스 번호 검색
3. 지하철역 검색
4. 환승 정보 조회
5. 주변 편의시설 조회
6. 즐겨찾기 조회
7. 즐겨찾기 추가
8. 최근 검색 기록
0. 종료
```

### 1-3. 프로젝트의 개발 개요

#### 시스템 구성도

○ 전체 시스템 구성

－ 본 시스템은 서버, 클라이언트, 공통 모듈, 데이터 파일, 사용자 로컬 저장 파일로 구성된다.

● 서버는 CSV 데이터를 메모리에 적재하고 검색 요청을 처리하며, 클라이언트는 콘솔 메뉴를 통해 사용자의 입력을 받아 서버와 통신한다.

```text
+----------------------------------------------------------+
|                          서버                            |
|  +------------+   +---------------+   +----------------+ |
|  | 접속 대기  |   | 클라이언트    |   | 요청 라우터    | |
|  | 루프       |-> | 처리 모듈     |-> |                | |
|  +------------+   +---------------+   +----------------+ |
|                              |                           |
|                              v                           |
|  +----------------------------------------------------+  |
|  | 검색 및 조회 서비스                                |  |
|  | 정류장 검색 | 버스 노선 검색 | 지하철역 검색       |  |
|  | 환승 경로 탐색 | 주변 편의시설 추천                |  |
|  | 도착 예정시간/운행 상태/혼잡도 계산                |  |
|  +-------------------------+--------------------------+  |
|                            |                             |
|  +-------------------------v--------------------------+  |
|  | 인메모리 데이터 저장소                              |  |
|  | 연결리스트: 정류장, 노선, 편의시설                 |  |
|  | 그래프: 환승 네트워크                              |  |
|  +-------------------------+--------------------------+  |
|                            |                             |
|  +-------------------------v--------------------------+  |
|  | CSV 로더 / 파일 기록 모듈                           |  |
|  | data/*.csv | logs/server.log                        |  |
|  +----------------------------------------------------+  |
+----------------------------------------------------------+
                  ^ TCP 소켓 ^
                  |            |
+----------------------------------------------------------+
|                       클라이언트                         |
|  +----------------+   +----------------+                 |
|  | 콘솔 메뉴      |-> | 요청 생성기     |                |
|  +----------------+   +----------------+                 |
|           ^                    |                          |
|           |                    v                          |
|  +----------------+   +----------------+                 |
|  | 결과 출력기    |<- | 소켓 통신 모듈  |                 |
|  +----------------+   +----------------+                 |
|           |                                               |
|           v                                               |
|  최근 검색 기록 파일 | 즐겨찾기 파일                     |
+----------------------------------------------------------+
```

○ 서버 처리 흐름

－ 서버는 실행 시 `data/` 폴더의 CSV 파일을 읽고, 정류장, 노선, 지하철역, 편의시설 데이터를 연결리스트와 그래프 자료구조에 저장한다.

● 이후 클라이언트 요청 타입에 따라 검색 서비스, 환승 서비스, 편의시설 서비스, 운행 정보 서비스로 분기하여 결과를 생성한다.

○ 클라이언트 처리 흐름

－ 클라이언트는 사용자의 메뉴 선택과 입력값을 검증한 뒤 서버 프로토콜에 맞는 요청 메시지를 생성한다.

● 서버 응답은 사용자가 바로 이해할 수 있도록 정류장 목록, 경유 버스, 환승 경로, 편의시설, 도착 예정시간, 혼잡도 등의 형태로 출력한다.

○ 메시지 프로토콜

－ 서버와 클라이언트는 `<TYPE>|<PAYLOAD>\n` 형식의 텍스트 기반 메시지를 사용한다.

● 주요 요청 타입은 `STOP_SEARCH_REQ`, `BUS_SEARCH_REQ`, `SUBWAY_SEARCH_REQ`, `TRANSFER_REQ`, `FACILITY_REQ`, `ARRIVAL_REQ`, `CROWDING_REQ`로 구성한다.

### 1-4. 활용방안 및 기대효과

○ 활용방안

－ 사용자는 로컬 또는 같은 네트워크 환경에서 서버와 클라이언트를 실행하여 아산시 대중교통 정보를 조회할 수 있다.

● 정류장명, 버스 번호, 지하철역명을 입력해 필요한 정보를 확인하고, 자주 사용하는 항목은 즐겨찾기에 저장하여 반복 조회 시간을 줄일 수 있다.

○ 사용자 편의성 향상

－ 정류장, 버스, 지하철, 환승, 편의시설 정보를 하나의 메뉴에서 조회할 수 있다.

● 낯선 지역에서 이동 계획을 세우는 사용자가 여러 정보를 따로 찾지 않고 프로그램 안에서 필요한 교통 정보를 확인할 수 있다.

○ 데이터 관리 및 확장성 확보

－ CSV 기반 데이터 구조를 사용하므로 교통 데이터 갱신과 테스트 데이터 추가가 비교적 쉽다.

● 향후 실시간 API, 지도 기반 UI, GUI 클라이언트 등을 추가하는 방향으로 확장할 수 있다.

○ 운영 확인성 확보

－ 서버 로그를 통해 요청 처리 내역, 오류, 접속 정보를 확인할 수 있다.

● 기능 시연 중 문제가 발생하더라도 로그를 기반으로 원인을 추적하고 수정할 수 있다.

## 2. 프로젝트 개발 상세 계획

### 2-1. 주요 구성요소 상세 개발 계획

#### 서버

○ 서버 실행 및 접속 관리

－ 서버는 지정된 포트에서 클라이언트 접속을 대기하고, 접속한 클라이언트의 요청을 수신한다.

● 포트가 이미 사용 중이거나 소켓 생성에 실패한 경우 오류 메시지를 출력하고, 정상 접속 시 클라이언트 IP와 접속 시간을 로그로 남긴다.

○ 요청 라우팅

－ 서버는 클라이언트가 보낸 요청 타입을 확인하여 기능별 처리 모듈로 전달한다.

● 알 수 없는 요청 타입, 필드 개수 오류, 빈 검색어는 `INVALID_REQUEST` 또는 `NOT_FOUND` 응답으로 처리하여 서버가 종료되지 않도록 한다.

○ CSV 데이터 로딩

－ 서버 시작 시 정류장, 버스 노선, 노선별 정류장, 지하철역, 편의시설, 도착 예정시간, 혼잡도 CSV 파일을 읽는다.

● 필수 컬럼 누락, 빈 ID, 중복 ID, 잘못된 참조는 로딩 단계에서 확인하고 서버 로그에 기록한다.

○ 검색 서비스

－ 정류장명, 버스 번호, 지하철역명 기반 검색 기능을 구현한다.

● 정류장 검색은 경유 버스 목록을 함께 제공하고, 버스 번호 검색은 출발지, 종점, 경유 정류장 순서를 출력하며, 지하철역 검색은 노선과 환승 가능 여부를 표시한다.

○ 환승 및 편의시설 서비스

－ 정류장과 역을 그래프 노드로 구성하고, 이동 가능 구간을 간선으로 연결하여 환승 경로를 탐색한다.

● 편의시설은 `nearest_node_id`와 카테고리를 기준으로 조회하며, 해당 위치에 데이터가 없을 경우 결과 없음 메시지를 반환한다.

○ 운행 정보 및 로그 관리

－ 테스트 CSV를 기준으로 도착 예정시간, 운행 상태, 혼잡도를 조회하거나 계산한다.

● 서버 로그에는 클라이언트 접속, 요청 타입, 처리 결과, 오류 정보를 기록하고 `logs/server.log`에 저장한다.

#### 클라이언트

○ 콘솔 메뉴 제공

－ 클라이언트는 사용자가 기능을 쉽게 선택할 수 있도록 1번부터 8번까지의 기능 메뉴와 0번 종료 메뉴를 제공한다.

● 잘못된 메뉴 번호가 입력되면 오류 안내 후 메뉴로 복귀한다.

○ 입력값 검증 및 요청 생성

－ 사용자의 검색어, 버스 번호, 출발지, 도착지, 카테고리 입력값을 검증한 뒤 서버 요청 메시지를 생성한다.

● 빈 입력, 너무 긴 입력, 형식이 맞지 않는 입력은 서버에 보내기 전에 클라이언트에서 먼저 안내한다.

○ 서버 통신 및 결과 출력

－ 클라이언트는 서버 IP와 포트로 접속하고 요청/응답을 송수신한다.

● 서버 응답은 검색 결과, 환승 경로, 편의시설, 도착 예정시간, 혼잡도, 오류 메시지 등 사용자에게 필요한 형태로 정리하여 출력한다.

○ 즐겨찾기 관리

－ 정류장, 버스 번호, 지하철역 정보를 `client_data/favorites.txt`에 저장한다.

● 동일한 유형과 ID의 항목이 이미 저장되어 있으면 중복 저장하지 않고 안내 메시지를 출력한다.

○ 최근 검색 기록 관리

－ 검색 유형, 키워드, 검색 시간을 `client_data/recent_search.txt`에 저장한다.

● 최근 검색 기록 메뉴에서는 저장된 검색 이력을 최신순으로 확인할 수 있도록 한다.

#### 공통 모듈

○ 프로토콜 정의

－ 요청/응답 타입, 오류 코드, 패킷 최대 길이, 구분자를 공통 헤더에 정의한다.

● 서버와 클라이언트가 동일한 메시지 규칙을 사용하도록 하여 요청 해석 오류를 줄인다.

○ 네트워크 호환 처리

－ Windows와 Linux의 소켓 초기화, 종료, 오류 처리 차이를 공통 모듈에서 감싼다.

● 플랫폼별 코드를 한 곳에 모아 서버와 클라이언트 구현부의 복잡도를 줄인다.

○ CSV 및 문자열 처리

－ CSV 라인을 필드 단위로 분리하고 필수값 누락 여부를 확인한다.

● 문자열 개행 제거, 부분 문자열 검색, 입력 길이 제한, 토큰 분리 함수를 제공한다.

○ 자료구조 처리

－ 연결리스트 노드 추가, 검색, 순회, 메모리 해제 함수를 구현한다.

● 그래프 노드 추가, 간선 추가, 경로 탐색 함수를 제공하여 환승 정보 조회에 활용한다.

### 2-2. 프로젝트 추진 계획

○ 개발 추진 방향

－ 먼저 실행 가능한 서버/클라이언트 연결과 CSV 로딩 구조를 완성한 뒤, 핵심 검색 기능을 구현한다.

● 이후 환승 경로, 편의시설 추천, 즐겨찾기, 최근 검색, 운행 정보 표시, 로그 및 오류 처리를 통합하여 사용자가 처음부터 끝까지 사용할 수 있는 완성형 프로그램으로 정리한다.

○ 개발 우선순위

－ 전체 기능은 기본 구조, 핵심 검색, 확장 조회, 사용자 편의, 운행 정보 표시 순서로 개발한다.

● 서버/클라이언트 연결과 데이터 로딩이 완료되어야 나머지 검색 및 조회 기능을 안정적으로 구현할 수 있다.

| 우선순위 | 요구사항 범위 | 개발 내용 | 핵심 이유 |
|---|---|---|---|
| P0 | FR-N01~N03, FR-DATA01~DATA03 | 서버/클라이언트 연결, CSV 로드, 연결리스트 및 그래프 기본 구조 | 프로그램 실행과 데이터 처리의 기반 |
| P1 | FR-S01~S03 | 정류장 검색, 버스 번호 검색, 지하철역 검색 | 사용자가 가장 먼저 사용하는 핵심 조회 기능 |
| P2 | FR-S04~S05, FR-DATA05 | 환승 정보 조회, 주변 편의시설 조회, 테스트 데이터 추가 | 이동 의사결정에 필요한 확장 기능 |
| P3 | FR-U01~U06, FR-N04 | 즐겨찾기, 최근 검색 기록, 메뉴 UI, 서버 로그 | 반복 사용성과 운영 확인성 확보 |
| P4 | FR-O01~O04 | 도착 예정시간, 혼잡도, 운행 상태 표시 | 교통 정보 프로그램으로서의 완성도 향상 |
