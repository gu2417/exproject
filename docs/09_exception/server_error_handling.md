# 서버 오류 처리

## 1. 서버 시작 단계

| 상황 | 처리 |
|------|------|
| 포트 사용 중 | 오류 출력 후 종료 |
| 필수 CSV 없음 | `DATA_LOAD_ERROR` 로그 후 종료 또는 기능 제한 |
| CSV 필수 컬럼 누락 | 파일명과 컬럼명 로그 기록 |
| 메모리 할당 실패 | 로딩 중단, 이미 할당한 메모리 해제 |

## 2. 요청 처리 단계

| 상황 | 응답 |
|------|------|
| 알 수 없는 패킷 타입 | `ERROR|2:INVALID_REQUEST` |
| 필드 부족 | `ERROR|2:INVALID_REQUEST` |
| 검색 결과 없음 | `<TYPE>_RES|1|` |
| 내부 오류 | `ERROR|4:SERVER_ERROR` |

## 3. 연결 단계

- `recv()`가 0을 반환하면 정상 종료로 처리한다.
- `recv()`가 오류를 반환하면 오류 로그 후 세션을 정리한다.
- 클라이언트별 처리 종료 시 소켓을 반드시 닫는다.
