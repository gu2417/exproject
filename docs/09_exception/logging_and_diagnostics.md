# 로그 및 진단

## 1. 서버 로그 형식

```text
YYYY-MM-DD HH:MM:SS | <client_ip> | <request_type> | <result_code> | <message>
```

## 2. 로그 예시

```text
2026-05-21 14:30:00 | 127.0.0.1 | CONNECT | 0 | client connected
2026-05-21 14:30:05 | 127.0.0.1 | STOP_SEARCH_REQ | 0 | keyword=온양 count=3
2026-05-21 14:31:10 | 127.0.0.1 | BUS_SEARCH_REQ | 1 | route_no=9999 not found
2026-05-21 14:32:00 | 127.0.0.1 | DISCONNECT | 0 | client closed
```

## 3. 반드시 남길 이벤트

| 이벤트 | 로그 필요 |
|--------|-----------|
| 서버 시작/종료 | Y |
| CSV 로딩 성공/실패 | Y |
| 클라이언트 접속/종료 | Y |
| 요청 타입과 결과 코드 | Y |
| 데이터 검증 오류 | Y |
| 내부 오류 | Y |

## 4. 진단 절차

1. 서버가 시작되지 않으면 포트와 CSV 로딩 로그를 확인한다.
2. 검색 결과가 없으면 요청 타입, 키워드, 데이터 파일 ID를 확인한다.
3. 환승 결과가 없으면 그래프 노드 생성 로그와 참조 오류를 확인한다.
4. 클라이언트가 종료되면 네트워크 오류와 서버 disconnect 로그를 확인한다.
