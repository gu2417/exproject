# 인덱스 및 성능 전략

DB 인덱스는 없지만, 인메모리 구조에서 반복 검색 비용을 줄이는 보조 구조를 둘 수 있다.

## 1. 기본 검색

| 기능 | 기본 방식 | 예상 비용 |
|------|-----------|-----------|
| 정류장 검색 | 연결리스트 순회 | O(N) |
| 버스 번호 검색 | 노선 연결리스트 순회 | O(R) |
| 지하철역 검색 | 역 연결리스트 순회 | O(S) |
| 편의시설 검색 | 시설 연결리스트 순회 | O(F) |
| 환승 탐색 | 가중치 기반 최단 경로 | O(V^2+E) |

## 2. 보조 인덱스 후보

구현 시간이 허용되면 다음 보조 구조를 추가할 수 있다.

| 인덱스 | 목적 |
|--------|------|
| route_no -> RouteNode 포인터 배열/해시 | 버스 번호 검색 가속 |
| stop_id -> StopNode 포인터 배열/해시 | 노선 정류장 이름 변환 가속 |
| node_id -> graph index | 환승 출발/도착 노드 해석 가속 |

## 3. 성능 원칙

- CSV는 서버 시작 시 한 번만 읽는다.
- 검색 요청 중 파일을 다시 읽지 않는다.
- 응답 문자열 생성 시 버퍼 길이를 항상 확인한다.
- 불필요하게 전체 구조를 복사하지 않고 포인터 순회 중심으로 처리한다.
