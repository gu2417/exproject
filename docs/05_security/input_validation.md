# 입력 검증

## 1. 검증 대상

| 입력원 | 검증 항목 |
|--------|-----------|
| 콘솔 메뉴 | 숫자 여부, 메뉴 범위 |
| 검색어 | 빈 문자열, 최대 길이, 제어문자 |
| 패킷 | `TYPE`, 필드 개수, 최대 길이, 종단 문자 |
| CSV | 필수 컬럼, 빈 ID, 숫자 필드, 중복 ID |
| 로컬 파일 | 구분자 개수, 필드 길이, 손상된 행 |

## 2. 패킷 검증 규칙

1. `\n` 기준으로 한 패킷을 완성한다.
2. 전체 길이가 `MAX_PACKET_SIZE`를 넘으면 거부한다.
3. `TYPE`은 대문자, 숫자, `_`만 허용한다.
4. 요청 타입별 필수 필드 수를 확인한다.
5. 검색어가 비어 있으면 `INVALID_REQUEST`로 처리한다.

## 3. 문자열 길이 기준

| 값 | 권장 최대 길이 |
|----|----------------|
| 정류장명/역명 검색어 | 100 bytes |
| 버스 번호 | 32 bytes |
| 시설 카테고리 | 50 bytes |
| 주소 | 200 bytes |
| 패킷 전체 | 4096 bytes |

## 4. 실패 처리

검증 실패는 서버 크래시가 아니라 오류 응답과 로그로 처리한다.

```text
ERROR|2:INVALID_REQUEST
```
