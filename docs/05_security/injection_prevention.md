# 삽입 및 조작 방지

## 1. 프로토콜 구분자 삽입

검색어와 로컬 파일 필드에는 `|`, `:`, `;`, `\n`이 들어올 수 있다.
구분자가 포함된 입력은 요청별 정책에 따라 거부하거나 화면 출력용으로만 escape한다.

| 문자 | 정책 |
|------|------|
| `|` | 패킷 필드 구분자이므로 일반 입력에서 거부 |
| `:` | 내부 항목 구분자이므로 ID/키워드 입력에서 거부 권장 |
| `;` | 리스트 구분자이므로 로컬 저장 필드에서 거부 |
| `\n`, `\r` | 패킷 경계 및 로그 위조 방지를 위해 제거 또는 거부 |

## 2. 로그 위조 방지

- 로그에 기록하는 사용자 입력은 개행 문자를 제거한다.
- 긴 입력은 최대 길이까지만 기록한다.
- 로그 형식은 `timestamp | ip | type | result | message`처럼 고정한다.

## 3. 파일 경로 조작 방지

사용자가 임의 파일 경로를 입력해 CSV 또는 로그 경로를 바꾸지 않게 한다.
프로그램이 사용하는 파일은 고정된 상대 경로만 허용한다.

```text
data/
client_data/
logs/
```

## 4. CSV 파싱 안정성

- 필드 개수가 예상보다 적은 행은 건너뛴다.
- 숫자 변환 실패는 해당 행 오류로 기록한다.
- 문자열 복사는 `safe_strncpy()` 같은 길이 제한 함수를 사용한다.
