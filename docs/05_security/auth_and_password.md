# 인증 및 비밀번호

## 1. 현재 범위

이 프로젝트는 사용자 계정, 로그인, 비밀번호를 다루지 않는다.
클라이언트는 서버 IP와 포트로 접속하며, 인증 없이 메뉴 기능을 사용한다.

## 2. 범위 밖 항목

| 항목 | 상태 |
|------|------|
| 회원가입 | 구현하지 않음 |
| 로그인 | 구현하지 않음 |
| 비밀번호 저장 | 구현하지 않음 |
| 사용자별 서버 저장소 | 구현하지 않음 |

## 3. 기본 접속 안전선

- 서버는 로컬 또는 같은 학습 네트워크 환경에서 사용하는 것을 전제로 한다.
- 외부 공개망에서 실행할 경우 방화벽과 포트 제한이 필요하다.
- 인증이 없으므로 중요한 개인정보를 패킷이나 로그에 넣지 않는다.

## 4. 향후 확장 시 고려사항

계정 기능이 추가된다면 비밀번호 평문 저장은 금지하고, 해시와 salt를 적용한다.
단, 현재 요구사항에는 포함하지 않는다.
