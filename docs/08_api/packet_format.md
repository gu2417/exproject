# 패킷 포맷

## 1. 기본 형식

```text
<TYPE>|<PAYLOAD>\n
```

또는 응답별로 여러 필드를 사용할 수 있다.

```text
<TYPE>|<code>|<payload>\n
```

## 2. 구분자

| 용도 | 구분자 | 예 |
|------|--------|----|
| 필드 | `|` | `STOP_SEARCH_REQ|온양` |
| 항목 내부 | `:` | `STOP001:온양온천역:100,101` |
| 리스트 항목 | `;` | `item1;item2;item3` |
| 패킷 종료 | `\n` | 한 줄 단위 송수신 |

## 3. 크기와 인코딩

| 항목 | 값 |
|------|----|
| 최대 패킷 길이 | 4096 bytes |
| 문자열 인코딩 | UTF-8 |
| TYPE 길이 | 32 bytes 이하 권장 |

## 4. 수신 검증

1. `\n`을 만날 때까지 버퍼에 누적한다.
2. 4096 bytes를 초과하면 연결을 종료하거나 `INVALID_REQUEST`로 처리한다.
3. `TYPE`이 비어 있거나 알 수 없는 값이면 오류 응답을 반환한다.
4. 요청별 필드 수가 부족하면 `INVALID_REQUEST`로 처리한다.
