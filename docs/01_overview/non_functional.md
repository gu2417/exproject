# 비기능 요구사항

## NFR-01 소켓 통신

TCP 기반 서버/클라이언트 구조로 요청과 응답을 처리한다. 서버는 지정 포트에서 대기하고,
클라이언트는 서버 IP와 포트로 접속한다.

## NFR-02 응답 시간

일반 검색은 로컬 테스트 환경에서 1초 이내 응답을 목표로 한다. 서버 시작 시 CSV 데이터를
메모리에 로드하여 매 요청마다 파일을 다시 읽지 않는다.

## NFR-03 안정성

빈 검색어, 잘못된 메뉴 번호, 알 수 없는 패킷 타입, 클라이언트 비정상 종료가 발생해도
서버 프로세스가 종료되지 않아야 한다.

## NFR-04 파일 입출력

공공데이터 CSV, 테스트 CSV, 서버 로그, 즐겨찾기, 최근 검색 기록을 파일로 읽고 쓴다.
파일 열기 실패는 명확한 오류와 로그로 처리한다.

## NFR-05 자료구조 요구

정류장, 노선, 편의시설 등 검색 데이터는 직접 구현한 연결리스트로 저장한다.
환승 기능에는 그래프 자료구조를 적용한다.

## NFR-06 이식성

Windows와 Linux에서 빌드 가능한 C11 코드 구성을 목표로 한다. 플랫폼별 소켓 초기화와
종료 처리는 공통 네트워크 호환 계층으로 분리한다.

## NFR-07 데이터 품질

CSV 컬럼 누락, 빈 ID, 중복 ID, 잘못된 노선 참조를 로딩 시 검증한다. 오류 데이터는
건너뛰거나 로딩 실패로 처리하고 로그에 남긴다.

## NFR-08 로그 추적성

서버 로그에는 시간, 클라이언트 IP, 요청 타입, 처리 결과, 오류 메시지를 기록한다.
시연 중 장애가 발생하면 `logs/server.log`를 통해 원인을 추적할 수 있어야 한다.

## NFR-09 한글 처리

CSV, 콘솔 출력, 프로토콜 문자열은 UTF-8 기준으로 처리한다. 한글 검색어가 잘리지 않도록
버퍼 길이와 문자열 복사를 안전하게 처리한다.

## NFR-10 범위 명확성

실시간 API 없이 정적 CSV 및 테스트 데이터 기반으로 기능을 구현한다. UI와 문서에는
실시간 정확도 보장이 범위 밖임을 명확히 한다.
