# 목적 및 범위

## 1. 프로젝트 목적

**아산시 대중교통 정보 시스템**을 C11 기반 TCP 서버/클라이언트 프로그램으로 구현한다.
사용자는 콘솔 클라이언트에서 정류장, 버스 번호, 지하철역, 환승 경로, 주변 편의시설,
도착 예정시간, 혼잡도, 운행 상태를 조회할 수 있다.

서버는 공공데이터포털에서 확보한 CSV 파일을 시작 시 로드하고, 연결리스트와 그래프
자료구조를 이용해 검색 요청을 처리한다.

## 2. 대상 사용자

- 아산시 버스/지하철 정보를 빠르게 조회하려는 일반 사용자.
- C 소켓 프로그래밍, 파일 입출력, 자료구조 적용 결과를 확인해야 하는 평가자.
- 로컬 또는 같은 네트워크 환경에서 서버와 클라이언트를 실행하는 개발·시연 사용자.

## 3. In-Scope

- TCP Server-Client 분리 구조.
- 콘솔 메뉴 기반 클라이언트.
- 공공데이터 CSV 로딩 및 파일 입출력.
- 정류장, 버스 노선, 지하철역, 편의시설 검색.
- 연결리스트 직접 구현 및 검색 데이터 저장.
- 그래프 자료구조 기반 환승 경로 탐색.
- 즐겨찾기와 최근 검색 기록의 클라이언트 로컬 파일 저장.
- 서버 접속, 요청, 응답, 오류 로그 저장.
- 테스트 CSV 기반 도착 예정시간, 운행 상태, 혼잡도 표시.
- Windows와 Linux에서 빌드 가능한 C11 코드 구성.

## 4. Out-of-Scope

- 채팅 기능.
- 학생정보관리 기능.
- 실시간 교통 API 연동.
- 정확한 실시간 차량 위치 추적.
- GUI 애플리케이션.
- 모바일 앱, 웹 클라이언트, 지도 기반 UI.

## 5. 성공 기준

1. 서버와 클라이언트가 TCP로 연결되고, 메뉴 선택에 따라 요청/응답을 수행한다.
2. 서버 시작 시 CSV 데이터를 읽고 연결리스트 및 그래프 구조에 저장한다.
3. 정류장, 버스 번호, 지하철역 검색이 정상 동작한다.
4. 환승, 편의시설, 도착 예정시간, 혼잡도, 운행 상태가 테스트 데이터 기반으로 표시된다.
5. 즐겨찾기, 최근 검색 기록, 서버 로그가 파일로 저장된다.
6. 잘못된 입력, 결과 없음, 파일 누락, 클라이언트 종료 상황에서도 서버가 크래시하지 않는다.
