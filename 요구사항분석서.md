# 소켓 기반 아산시 대중교통 및 주변 편의시설 정보 조회 시스템 요구사항 분석서

## 1. 프로젝트 개요

본 프로젝트는 C언어 기반 콘솔 프로그램으로 구현하는 **아산시 대중교통 및 주변 편의시설 정보 조회 시스템**이다.  
서버와 클라이언트를 분리하여 TCP 소켓 통신을 사용하며, 클라이언트는 메뉴 기반 콘솔 UI를 통해 정류장, 버스, 지하철역, 환승 정보, 주변 편의시설 등을 조회한다.

시스템은 실시간 API를 연동하지 않고, CSV 및 TXT 파일에 저장된 테스트용 공공데이터 기반 데이터를 읽어 동작한다. 도착 예정 시간과 혼잡도는 실제 API 값이 아니라 CSV에 포함된 테스트 값 또는 서버에서 생성한 값으로 제공한다.

## 2. 개발 목적 및 필요성

아산시 내 버스, 지하철역, 환승 지점, 주변 편의시설 정보를 한 번에 조회할 수 있는 간단한 콘솔 기반 정보 시스템을 구현하는 것이 목적이다.

본 프로젝트를 통해 다음 내용을 학습하고 적용한다.

- C언어 기반 파일 입출력
- Winsock 기반 TCP 소켓 프로그래밍
- 서버/클라이언트 구조 설계
- 연결리스트, 큐, 그래프 자료구조 활용
- CSV 데이터를 이용한 검색 기능 구현
- 메뉴 기반 콘솔 UI 구성
- 로그 및 사용자 기록 파일 관리

## 4. 시스템 구성

시스템은 서버 프로그램과 클라이언트 프로그램으로 분리한다.

### 4.1 서버

서버는 실행 시 `data` 폴더의 CSV 파일을 읽어 메모리 자료구조를 초기화한다. 이후 클라이언트 접속을 기다리고, 클라이언트가 보낸 요청 문자열을 분석하여 검색 결과를 반환한다.

서버의 주요 역할은 다음과 같다.

- CSV 파일 로드
- 연결리스트 및 그래프 초기화
- 클라이언트 TCP 접속 대기
- 요청 문자열 파싱
- 검색, 환승, 편의시설, 즐겨찾기, 최근 검색 처리
- 처리 결과 문자열 생성 및 전송
- 요청 로그 저장

### 4.2 클라이언트

클라이언트는 사용자가 직접 실행하는 메뉴 기반 콘솔 프로그램이다. 사용자는 메뉴 번호를 선택하고 검색어를 입력하며, 클라이언트는 이를 서버 요청 메시지 형식으로 변환하여 서버에 전송한다.

클라이언트 메뉴는 다음과 같이 구성한다.

```text
===== 아산시 대중교통 정보 시스템 =====
1. 정류장 검색
2. 버스 번호 검색
3. 지하철역 검색
4. 환승 정보 조회
5. 주변 편의시설 조회
6. 즐겨찾기 조회
7. 즐겨찾기 추가
8. 최근 검색 기록
0. 종료
```

## 5. 기능 요구사항

### 5.1 정류장 검색

사용자가 정류장 이름을 입력하면 해당 정류장에 정차하는 버스 목록을 조회한다.

출력 정보는 다음 항목을 포함한다.

- 정류장명
- 버스 번호
- 도착 예정 시간
- 혼잡도
- 운행 상태

출력 예시는 다음과 같다.

```text
정류장명: 온양온천역
101번 버스 / 도착 예정: 3분 후 / 혼잡도: 보통 / 운행 상태: 곧 도착
900번 버스 / 도착 예정: 8분 후 / 혼잡도: 여유 / 운행 상태: 운행중
```

### 5.2 버스 번호 검색

사용자가 버스 번호를 입력하면 해당 버스가 지나가는 정류장을 노선 순서대로 출력한다.

출력 예시는 다음과 같다.

```text
101번 버스 노선
온양온천역 -> 아산터미널 -> 배방역 -> 천안아산역
```

### 5.3 지하철역 검색

사용자가 지하철역 이름을 입력하면 해당 역의 노선명과 인근 버스 정류장 정보를 출력한다.

조회 데이터는 `subway_data.csv` 파일을 기준으로 한다.

### 5.4 즐겨찾기 기능

사용자는 정류장 또는 버스 번호를 즐겨찾기에 저장할 수 있다. 즐겨찾기 데이터는 `user/favorites.txt` 파일에 저장한다.

즐겨찾기 기능은 다음 동작을 포함한다.

- 정류장 즐겨찾기 추가
- 버스 번호 즐겨찾기 추가
- 즐겨찾기 목록 조회
- 즐겨찾기 목록을 이름 또는 번호 기준으로 정렬 출력
- 즐겨찾기 항목 선택 후 바로 검색

### 5.5 로그 저장

서버는 클라이언트 요청 처리 내역을 `logs/server_log.txt` 파일에 저장한다.

로그에는 다음 항목을 포함한다.

- 요청 시간
- 요청 종류
- 검색어
- 처리 결과

로그 예시는 다음과 같다.

```text
[2026-05-20 14:30] 정류장 검색 요청: 온양온천역 / 결과: 3건 반환
```

### 5.6 운행 상태 표시

버스 검색 결과에는 운행 상태를 표시한다. 운행 상태는 실제 API 값이 아니라 도착 예정 시간 또는 시간대 기준으로 서버에서 생성한다.

운행 상태 값은 다음 중 하나를 사용한다.

- 운행 시작
- 곧 도착
- 운행중
- 운행 종료

### 5.7 도착 예정 시간 표시

실시간 API를 사용하지 않으므로 도착 예정 시간은 테스트용 값으로 처리한다.

도착 예정 시간은 다음 방식 중 하나로 제공한다.

- `bus_data.csv`의 `arrival_time` 컬럼 값 사용
- 서버에서 1~15분 사이의 테스트 값을 생성

검색 결과에는 도착 예정 시간이 반드시 포함되어야 한다.

### 5.8 환승 시스템

사용자가 출발지와 도착지를 입력하면 버스 정류장과 지하철역 사이의 환승 경로를 출력한다.

환승 시스템은 그래프 자료구조를 사용하며, BFS 방식으로 경로를 탐색한다.

출력 예시는 다음과 같다.

```text
출발지: 온양온천역
도착지: 천안아산역
추천 경로: 온양온천역 -> 아산역 -> 천안아산역
환승 지점: 아산역
```

### 5.9 주변 편의시설 추천

사용자가 정류장 또는 지하철역 이름을 입력하면 해당 위치 주변 편의시설을 추천한다.

대상 시설은 다음과 같다.

- 병원
- PC방
- 편의점
- 약국

출력 예시는 다음과 같다.

```text
[배방역 주변 편의시설]
편의점: CU 배방역점, GS25 배방중앙점
약국: 배방약국
병원: 배방의원
PC방: OOO PC방
```

### 5.10 혼잡도 표시

혼잡도는 실제 API 값이 아니라 시간대별 기준으로 생성한다.

혼잡도 기준은 다음과 같다.

| 시간대 | 혼잡도 |
| --- | --- |
| 07:00~09:00 | 혼잡 |
| 12:00~14:00 | 보통 |
| 18:00~20:00 | 혼잡 |
| 그 외 시간 | 여유 |

정류장 검색 결과에는 혼잡도 값이 반드시 포함되어야 한다.

### 5.11 최근 검색 기록

사용자의 최근 검색 내역은 `user/recent_search.txt` 파일에 저장한다.

저장 항목은 다음과 같다.

- 검색 종류
- 검색어
- 검색 시간

클라이언트는 최근 검색 기록을 확인할 수 있는 메뉴를 제공하며, 이전 검색 내용을 다시 확인할 수 있도록 출력한다.

## 6. 비기능 요구사항

- 프로그램은 Windows 환경에서 실행 가능해야 한다.
- 서버와 클라이언트는 각각 별도 실행 파일로 컴파일되어야 한다.
- gcc로 컴파일 가능해야 한다.
- 콘솔 출력 문자열과 주석은 UTF-8 기준으로 작성한다.
- 코드 구조는 대학 프로젝트에서 설명 가능한 수준으로 단순하게 유지한다.
- 채팅 프로그램 또는 학생정보관리 프로그램처럼 보이지 않도록 대중교통 정보 조회 기능 중심으로 구현한다.
- 실시간 API 연동은 하지 않는다.
- 파일 열기 실패, 서버 연결 실패, 검색 결과 없음, 잘못된 메뉴 입력에 대한 기본 에러 처리를 포함한다.

## 7. 데이터 요구사항

### 7.1 CSV 데이터

CSV 샘플 데이터는 실제 공공데이터를 가공해서 사용하는 상황을 가정하여 작성한다. 프로젝트 테스트가 가능하도록 각 주요 데이터는 최소 10개 이상의 샘플을 포함한다.

#### bus_data.csv

```csv
bus_number,station_name,station_order,arrival_time
```

필드 설명은 다음과 같다.

| 필드 | 설명 |
| --- | --- |
| bus_number | 버스 번호 |
| station_name | 정류장 이름 |
| station_order | 노선 내 정류장 순서 |
| arrival_time | 테스트용 도착 예정 시간 |

#### subway_data.csv

```csv
station_name,line_name,near_bus_station
```

필드 설명은 다음과 같다.

| 필드 | 설명 |
| --- | --- |
| station_name | 지하철역 이름 |
| line_name | 노선명 |
| near_bus_station | 인근 버스 정류장 |

#### facility_data.csv

```csv
base_location,facility_type,facility_name,description
```

필드 설명은 다음과 같다.

| 필드 | 설명 |
| --- | --- |
| base_location | 기준 위치 |
| facility_type | 시설 종류 |
| facility_name | 시설명 |
| description | 시설 설명 |

#### transfer_data.csv

```csv
from_location,to_location,transfer_type
```

필드 설명은 다음과 같다.

| 필드 | 설명 |
| --- | --- |
| from_location | 출발 위치 |
| to_location | 도착 위치 |
| transfer_type | 연결 종류 |

### 7.2 TXT 데이터

| 파일 | 용도 |
| --- | --- |
| logs/server_log.txt | 서버 요청 처리 로그 저장 |
| user/favorites.txt | 즐겨찾기 목록 저장 |
| user/recent_search.txt | 최근 검색 기록 저장 |

## 8. 통신 프로토콜 요구사항

서버와 클라이언트는 단순 문자열 기반 요청 메시지를 사용한다. 각 항목은 `|` 문자로 구분한다.

| 기능 | 요청 메시지 형식 |
| --- | --- |
| 정류장 검색 | `STATION_SEARCH|온양온천역` |
| 버스 검색 | `BUS_SEARCH|101` |
| 지하철역 검색 | `SUBWAY_SEARCH|아산역` |
| 환승 정보 조회 | `TRANSFER|온양온천역|천안아산역` |
| 편의시설 조회 | `FACILITY|배방역` |
| 즐겨찾기 추가 | `FAVORITE_ADD|STATION|온양온천역` |
| 최근 검색 기록 조회 | `RECENT_LIST` |

응답 메시지는 사람이 읽을 수 있는 일반 문자열로 반환한다.

## 9. 자료구조 요구사항

### 9.1 연결리스트

연결리스트는 다음 데이터를 저장하고 검색하는 데 사용한다.

- 버스 데이터
- 정류장 데이터
- 지하철역 데이터
- 편의시설 데이터

### 9.2 큐

큐는 다음 기능에 사용한다.

- 도착 예정 순서 관리
- 검색 결과 출력 순서 관리
- 그래프 BFS 탐색 시 방문 대상 관리

### 9.3 그래프

그래프는 환승 시스템에서 정류장과 지하철역 사이의 연결 관계를 표현한다.

- 정점: 정류장 또는 지하철역
- 간선: 환승 또는 이동 가능 관계
- 탐색 방식: BFS

## 10. 파일 구조 요구사항

최종 프로젝트 파일 구조는 다음과 같이 구성한다.

```text
project/
|
|-- server.c
|-- client.c
|-- common.h
|-- data.h
|-- data.c
|-- linkedlist.h
|-- linkedlist.c
|-- queue.h
|-- queue.c
|-- graph.h
|-- graph.c
|-- fileio.h
|-- fileio.c
|-- favorite.h
|-- favorite.c
|-- log.h
|-- log.c
|
|-- data/
|   |-- bus_data.csv
|   |-- subway_data.csv
|   |-- facility_data.csv
|   `-- transfer_data.csv
|
|-- logs/
|   `-- server_log.txt
|
`-- user/
    |-- favorites.txt
    `-- recent_search.txt
```

각 파일의 역할은 다음과 같다.

| 파일 | 역할 |
| --- | --- |
| server.c | 서버 실행, 소켓 대기, 요청 처리 |
| client.c | 클라이언트 실행, 메뉴 UI, 요청 전송 |
| common.h | 공통 상수, 요청 코드, 버퍼 크기 정의 |
| data.h / data.c | 데이터 구조체 및 검색 로직 |
| linkedlist.h / linkedlist.c | 연결리스트 구현 |
| queue.h / queue.c | 큐 구현 |
| graph.h / graph.c | 그래프 및 BFS 환승 경로 탐색 |
| fileio.h / fileio.c | CSV 및 TXT 파일 입출력 |
| favorite.h / favorite.c | 즐겨찾기 저장, 조회, 정렬 |
| log.h / log.c | 서버 로그 저장 |

## 11. 예외 처리 요구사항

다음 상황에 대한 기본 예외 처리를 구현한다.

| 예외 상황 | 처리 방식 |
| --- | --- |
| CSV 파일 열기 실패 | 오류 메시지를 출력하고 서버 초기화 실패 처리 |
| TXT 파일 열기 실패 | 파일 생성 시도 또는 오류 메시지 출력 |
| 서버 연결 실패 | 클라이언트에서 연결 실패 메시지 출력 |
| 검색 결과 없음 | 결과가 없다는 안내 메시지 반환 |
| 잘못된 메뉴 입력 | 다시 입력하도록 안내 |
| 잘못된 요청 형식 | 서버에서 오류 응답 반환 |
| 소켓 통신 실패 | 연결 종료 또는 재시도 안내 |

## 12. 단계별 개발 계획

### 12.1 1단계 개발 범위

1단계에서는 전체 기능을 한 번에 완성하지 않고, 프로젝트의 기반 구조를 먼저 구현한다.

1단계 구현 대상은 다음과 같다.

- 프로젝트 폴더 및 파일 구조 생성
- 공통 헤더 `common.h` 작성
- 샘플 CSV 및 TXT 파일 작성
- Winsock 기반 서버 기본 구조 작성
- Winsock 기반 클라이언트 기본 구조 작성
- 서버와 클라이언트 간 기본 TCP 연결 확인
- 클라이언트 요청 문자열 전송
- 서버 응답 문자열 반환

### 12.2 2단계 개발 범위

2단계에서는 파일 입출력과 기본 검색 기능을 구현한다.

- CSV 파일 읽기
- 연결리스트에 버스, 지하철, 편의시설 데이터 저장
- 정류장 검색
- 버스 번호 검색
- 지하철역 검색
- 검색 결과 없음 처리

### 12.3 3단계 개발 범위

3단계에서는 사용자 편의 기능과 기록 기능을 구현한다.

- 즐겨찾기 추가
- 즐겨찾기 목록 조회
- 즐겨찾기 정렬 출력
- 최근 검색 기록 저장
- 최근 검색 기록 조회
- 서버 로그 저장

### 12.4 4단계 개발 범위

4단계에서는 환승 및 추천 기능을 구현한다.

- transfer_data.csv 기반 그래프 생성
- BFS 기반 환승 경로 탐색
- 주변 편의시설 추천
- 큐를 활용한 검색 결과 또는 BFS 순서 관리

### 12.5 5단계 개발 범위

5단계에서는 출력 품질과 테스트 안정성을 개선한다.

- 도착 예정 시간 표시
- 시간대별 혼잡도 표시
- 운행 상태 생성
- 메뉴 입력 검증
- 컴파일 및 실행 테스트
- 기능 테스트 예시 정리

## 13. 컴파일 및 실행 요구사항

### 13.1 컴파일 예시

Windows 환경에서 gcc를 사용하는 경우 다음과 같이 컴파일한다.

```bash
gcc server.c data.c linkedlist.c queue.c graph.c fileio.c favorite.c log.c -o server.exe -lws2_32
gcc client.c -o client.exe -lws2_32
```

### 13.2 실행 순서

먼저 서버를 실행한다.

```bash
server.exe
```

이후 다른 콘솔 창에서 클라이언트를 실행한다.

```bash
client.exe
```

## 14. 기능 테스트 시나리오

### 14.1 서버/클라이언트 연결 테스트

1. 서버 실행
2. 클라이언트 실행
3. 클라이언트에서 테스트 요청 전송
4. 서버가 응답 문자열을 반환하는지 확인

### 14.2 정류장 검색 테스트

입력:

```text
온양온천역
```

예상 결과:

```text
정류장명: 온양온천역
101번 버스 / 도착 예정: 3분 후 / 혼잡도: 보통 / 운행 상태: 곧 도착
900번 버스 / 도착 예정: 8분 후 / 혼잡도: 여유 / 운행 상태: 운행중
```

### 14.3 버스 번호 검색 테스트

입력:

```text
101
```

예상 결과:

```text
101번 버스 노선
온양온천역 -> 아산터미널 -> 배방역 -> 천안아산역
```

### 14.4 환승 정보 조회 테스트

입력:

```text
출발지: 온양온천역
도착지: 천안아산역
```

예상 결과:

```text
추천 경로: 온양온천역 -> 아산역 -> 천안아산역
환승 지점: 아산역
```

### 14.5 주변 편의시설 조회 테스트

입력:

```text
배방역
```

예상 결과:

```text
[배방역 주변 편의시설]
편의점: CU 배방역점, GS25 배방중앙점
약국: 배방약국
병원: 배방의원
PC방: OOO PC방
```

### 14.6 즐겨찾기 테스트

1. 정류장 `온양온천역` 즐겨찾기 추가
2. 버스 `101` 즐겨찾기 추가
3. 즐겨찾기 목록 조회
4. 이름 또는 번호 기준으로 정렬되어 출력되는지 확인
5. 즐겨찾기 항목 선택 후 바로 검색되는지 확인

### 14.7 최근 검색 기록 테스트

1. 정류장 검색 실행
2. 버스 번호 검색 실행
3. 최근 검색 기록 메뉴 선택
4. 검색 종류, 검색어, 검색 시간이 출력되는지 확인
