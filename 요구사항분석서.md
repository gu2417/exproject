## 3. 시스템 설계 요약

### 3-1. 프로젝트 개요

본 프로젝트는 아산시 버스 정류장, 버스 노선, 지하철역, 주변 편의시설 정보를 조회할 수 있는 대중교통 정보 시스템이다. 클라이언트는 콘솔 메뉴를 통해 요청을 보내고, 서버는 CSV 파일에서 로드한 교통 데이터를 기반으로 검색 결과를 반환한다.

프로젝트 범위는 채팅 또는 학생정보관리가 아니며, 공공데이터 CSV를 파일 입출력으로 읽어 사용하는 교통 정보 조회 시스템으로 제한한다.

### 3-2. 시스템 아키텍처

```text
----------------------+       TCP Socket       +----------------------+
|       CLIENT         | <--------------------> |        SERVER        |
| Console Menu         |                        | Accept Loop          |
| Request Builder      |                        | Client Handler       |
| Result Viewer        |                        | Command Router       |
| favorites.txt        |                        | Search Service       |
| recent_search.txt    |                        | In-Memory Data Store |
+----------------------+                        | CSV Loader / Logger  |
                                                +----------------------+
```

### 3-3. 메시지 프로토콜

```text
<TYPE>|<PAYLOAD>\n
```

| 항목 | 내용 |
|---|---|
| 필드 구분자 | `|` |
| 항목 내부 구분자 | `:` |
| 리스트 항목 구분자 | `;` |
| 종단 문자 | `\n` |
| 최대 패킷 길이 | 4096 bytes |
| 문자열 인코딩 | UTF-8 |

주요 요청 타입은 `STOP_SEARCH_REQ`, `BUS_SEARCH_REQ`, `SUBWAY_SEARCH_REQ`, `TRANSFER_REQ`, `FACILITY_REQ`, `ARRIVAL_REQ`, `CROWDING_REQ`로 구성한다.

### 3-4. 파일 기반 데이터 스키마

| 파일 | 주요 컬럼 | 설명 |
|---|---|---|
| `data/bus_stops.csv` | `stop_id, stop_name, lat, lng, district` | 아산시 버스 정류장 목록 |
| `data/bus_routes.csv` | `route_no, route_name, start_stop_id, end_stop_id, interval_min` | 버스 노선 기본 정보 |
| `data/route_stops.csv` | `route_no, seq, stop_id, direction` | 노선별 정류장 순서 |
| `data/subway_stations.csv` | `station_id, station_name, line_no, lat, lng, transfer_lines` | 지하철역 정보 |
| `data/facilities.csv` | `facility_id, name, category, nearest_node_id, address, lat, lng` | 정류장/역 주변 편의시설 |
| `data/arrival_test.csv` | `route_no, node_id, time_slot, arrival_minutes, status` | 도착 예정시간 및 운행 상태 테스트 데이터 |
| `data/crowding_test.csv` | `route_no, node_id, time_slot, crowding_level` | 시간대별 혼잡도 테스트 데이터 |

### 3-5. 클라이언트 콘솔 메뉴

```text
===== 아산시 대중교통 정보 시스템 =====

1. 정류장 검색
2. 버스 번호 검색
3. 지하철역 검색
4. 환승 정보 조회
5. 주변 편의시설 조회
6. 즐겨찾기 조회
7. 즐겨찾기 추가
8. 최근 검색 기록
0. 종료
```

### 3-6. 개발 우선순위

| 우선순위 | 요구사항 범위 | 핵심 이유 |
|---|---|---|
| P0 | FR-N01~N03, FR-DATA01~DATA03 | 서버/클라이언트 연결, CSV 로드, 자료구조 저장이 전체 기능의 기반 |
| P1 | FR-S01~S03 | 정류장, 버스, 지하철역 검색은 핵심 사용자 기능 |
| P2 | FR-S04~S05, FR-DATA05 | 환승 시스템과 편의시설 추천으로 프로젝트 차별화 |
| P3 | FR-U01~U06, FR-N04 | 즐겨찾기, 최근 검색, 로그 저장으로 사용성 및 검증 가능성 확보 |
| P4 | FR-O01~O04 | 도착 예정시간, 혼잡도, 운행 상태는 테스트 데이터 가공 후 완성 |
